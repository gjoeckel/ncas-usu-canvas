Tables present information in a structured format by arranging data into rows and columns. The format of a table makes it easy to read, compare, and understand information.

Why Accessible Tables Matter

Accessible tables ensure that students with and without disabilities can access information in equivalent ways. Formatting the table to be accessible helps students visually track and comprehend row and column data. It also ensures that a screen reader will announce table headers as students navigate the table with a keyboard, making it easier to interpret the data and follow the table's structure.

Learning Objectives

  1. Identify the characteristics of an accessible table.
  2. Describe why accessible tables are necessary for ensuring all students can understand your content.
  3. Apply best practices to create accessible tables.

Tips for Success

Use Clear Column and Row Headers

Headers act like labels for your table's data, helping all students, especially those using assistive technologies like screen readers, understand what each row and column represents. Always define headers in your table formatting tools to ensure that screen readers read the table content correctly.

Press play, then close your eyes and listen as a reader user describes their experience navigating a table formatted with proper row and column headers.

Keep Table Structures Simple

Avoid merged cells and mixing headers with data cells. A straightforward table layout helps students using assistive technology interpret data correctly and allows students to visually follow information more easily.

Avoid Using Images of Tables

Never use screenshots or pictures of tables. They are not readable by screen readers and can't be adjusted for accessibility. Instead, create tables using your content editor tools. For example, the tables on this page were built using the table tool in Canvas highlighted in the screenshot below. Once the table is inserted, Canvas's Accessibility Checker below the text editor box will prompt you to label the headers to improve accessibility.

Add a Descriptive Caption When Needed

When the purpose of the table isn't obvious from the context, include a caption or a short description outside the table. This helps all students understand what the table is showing, especially those with cognitive or visual disabilities.

Example: When a Caption Adds Meaning to Table Data

Table 1: Monthly Rainfall (in inches) Recorded in Three U.S. Cities During Spring 2024
Month Seattle Chicago Miami
March 5.2 3.4 2.9
April 3.9 3.8 3.2
May 2.7 4.5 5.1

If the content editor that you are using does not support table captions, a heading is a good alternative.

Use Tables to Organize Information, Not for Content Layout

Tables should organize data, not control where content appears on the page. Using tables for layout can confuse screen reader users and create barriers for students navigating with other assistive technology. Layout tables can also make it harder for students who zoom in or use mobile devices, as they may cause content to extend off-screen and require horizontal scrolling.

Use Visual Cues to Support Comprehension

Apply visual formatting, like bold text for headers or alternating row colors, to help students quickly scan and interpret the table. These visual cues enhance clarity for all students, especially those with learning or visual processing disabilities, but should always be used in addition to (not instead of) proper table header structure.

Accessible Tables Applied: Creating a Class Schedule

Mr. Hernandez wants his students to know when each of his 7th grade chemistry class units will be covered during the semester, including the topics and exam schedule for each one. The class is taught in four units with two exams per unit.

Mr. Hernandez creates a table in Microsoft Word. He makes "Topic" the header for the first column, "Unit" the header for the second, then "Exam 1" and "Exam 2" for the third and fourth columns. He puts the topics, unit numbers, and exam schedule in the appropriate columns. The topics are also headers for data in each row.

Topic Unit Exam 1 Exam 2
States of Matter 1 Week 2 Week 4
Atoms and Molecules 2 Week 6 Week 8
Chemical Reactions 3 Week 10 Week 12
The Periodic Table 4 Week 14 Week 17

Mr. Hernandez makes sure that the table is accessible by checking that the "Header Row" and "First Column" options are selected in the Table Design ribbon in Word.

Since Mr. Hernandez made the table accessible, Felix will hear changes in row or column headers as he navigates the table while using his screen reader. For example, when Felix moves between the cells that read "Week 2" and "Week 4" his screen reader will announce the column header for "Week 4": "Exam 2." The alternating row colors and bold column and row headers also make it easier for Mr. Hernandez's students to visually track and make sense of the information.
